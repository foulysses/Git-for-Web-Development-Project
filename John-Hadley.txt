1. What is Semantic HTML? 
    - HTML that introduces meaning to the web page rather than just presentation.
2. What is HTML used for?
    - Display data in a graphical form.  
3. What is an attribute and where do we put it?
    - HTML attributes provide additional information about HTML elements. The attribute is put after the opening tag.  
4. What is the h1 tag used for? How many times should I use it on a page?
    - Used to designate the title of a page or post. It should only be used once on a webpage. 
5. Name two tags that have required attributes?
    - <img> and <a>. 
6. What do we put in the head of our HTML document? 
    - Metadata.
7. What is an id?
    - An attribute used to identify elements. An id must always be unique to that single element. 
8. What elements can I add an id to?
    - An id can be added to any element.  
9. How many times can I use the same id on a page? 
    - Once
10. What is a class? 
    - An attribute used to identify elements. A class can be used for multiple elements on a page.
11. What elements can I add a class to? 
    - A class can be added to any element.
12. How many times can I use the same class on a page?
    - Unlimited 
13. How do I get my link to open in a new tab?
    - By adding the attribute target=”_blank” after the url.
14. What is the alt attribute in the image tag used for? 
    - Specifies an alternate text for an image if the image cannot be displayed. 
15. How do I reference an id?
    - #id{...}
16. What is the difference between a section and a div?
    - <div> does not contain any meaning and should only be used when no other semantic element is appropriate. <section> denotes a section of your document in a semantic way. 
17. What is CSS used for? 
    - Add style and layouts to webpages.
18. How do we select an element? Example - every h2 on the page
    - h2{...}
19. What is the difference between a class and an id? - Give me an example of when I might use each one
    - An id can only be used once per page but a class can be used multiple times. You might use an id to reference a name that only appears once on your page and you might use a class to reference the months of the year on your page.
20. How do we select classes in CSS?
    - .class{...}
21. How do we select a p element with a single class of “human””?
    - .human{...}
22. What is a parent child selector? When would this be useful? 
    - An element that is an immediate descendent of another element or nested within another element is called a child. These become useful when using CSS child selectors and psuedo-elements.
23. How do you select all links within a div with the class of sidebar?
    - .sidebar:link{...}
24. What is a pseudo selector?
    - A selector that selects elements that are in a specific state, e.g. they are the first element of their type, or they are being hovered over by the mouse pointer.
25. What do we use the change the spacing between lines?
    - line-height:...;
26. What do we use to change the spacing between letters?
    - letter-spacing:...;
27. What do we use to to change everything to CAPITALS? lowercase? Capitalize?
    - text-transform:uppercase; text-transform:lowercase; text-transform:capitalize;
28. How do I add a 1px border around my div that is dotted and black?
    - div{  border: 1px dotted black; }
29. How do I select everything on the page? 
    - body {...}
30. How do I write a comment in CSS?
    - /*...*/
31. How do I find out what file I am in, when I am using the command line?
    - By using the command pwd. 
32. Using the command line - how do I see a list of files/folders in my current folder?
    - By using the command ls.
33. How do I remove a file via the command line? Why do I have to be careful with this? 
    - By using the command rm, be careful with this because it is very difficult to undo.
34. Why should I use version control?
    - It helps you so you never lose any progress, all your work is saved in the version history.
35. How often should I commit to github?
    - Often and be sure to use meaningful commit messages. 
36. What is the command we would use to push our repo up to github? 
    - git push origin -u firstName-lastName
37. Walk me through Lambda's git flow.
    - FORK the repo on Github
    - CLONE in VS Code 
    - BRANCH in VS Code 
    - Make your changes and add your files
    - COMMIT your changes in VS Code 
    - PUSH your commit to Github in VS Code 
    - PULL to the main branch in Github
